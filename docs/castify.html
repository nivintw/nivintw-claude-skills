castify · documentation

Scriptable terminal recordings.

asciinema records a session you perform by hand. castify lets you script it — every keystroke and pause in a file — so a cast is reproducible (re-record after a UI change), deterministic (pacing lives in the script, not your typing), and automatable.

v0.2.0 documentation asciinema tmux fzf source ↗

Why tmux

The enabling trick is the payoff, too: castify drives the recording through a tmux pane with tmux send-keys, writing keystrokes into the session's real PTY. That is the only way to script interactive TUIs like fzf, less, or vim — they read the controlling TTY, not stdin, so you cannot pipe input to them.

Commands

One skill — it activates when you ask Claude Code to record a cast.

/castify:record-terminal-casts

Script a terminal session keystroke by keystroke and render it to a clean, embeddable asciinema cast — including interactive TUIs that can't be driven by piping stdin. Ships a reusable tmux recording library, a clean-shell recorder, a runnable worked example, and an end-to-end web-embedding guide.

try "record a terminal cast of this command" · "demo this CLI as a recording" · "record an fzf session"

Use it without Claude Code

The scripts stand alone. Requires tmux and asciinema.

# run the worked example, then verify the cast cd plugins/castify/skills/record-terminal-casts/scripts CAST_LAB=/tmp/castlab bash example-fixtures.sh CAST_OUT=./casts CAST_LAB=/tmp/castlab bash example-record.sh asciinema convert -f txt ./casts/fco.cast /dev/stdout

To put the resulting .cast files on a web page, see the embedding guide at skills/record-terminal-casts/reference/embedding.md — it vendors asciinema-player and covers licensing it correctly.
